Skiplink demo - 03 Fixing the problem in IE 5.5, 6, 7 and for Firefox

Another solution in IE instead of using HREF for anchors, is to use TABINDEX="-1" or any style the forces IE's haslayout value to be set (e.g. STYLE="FLOAT: LEFT;"). Latter needs additional markup (e.g. a SPAN) surrounding the anchor. In the case of using the SPAN element, navigation via ID does not work anymore for that element (try "skip to submenu" in Navigate via id).

Now the unnecessary step in Firefox during TAB navigation for the anchor is gone and everything works fine.

As expected the solution does also NOT work in Chrome.
