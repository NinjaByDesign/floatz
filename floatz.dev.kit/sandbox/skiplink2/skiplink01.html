Skiplink demo - 01 The problem space

Both navigation methods work in IE 5.5, 6, 7 to navigate within the content. But the problem is, while the screen scrolls to the correct item on the page, the keyboard focus is never moved to the item the user navigates. If TAB key is pressed, the keyboard navigation tarts again on the top of the page.

In Chrome, Opera and Safari the behaviour is similar to IE.

In Firefox the keyboard focus is also set as expected. Everything works fine.
