Skiplink demo - 02 Fixing the problem in IE 5.5, 6, 7

The main problem in IE seems to be the fact, that the target anchor element has no HREF value. After setting the HREF value, the focus is also moved to the corresponding section in the page. As tests showed, it is absolute ok if you use # as value for every skip link target anchor.

Whats really interesting is, that now the navigation via ids also works, although the target item is not an anchor with an HREF. Only the fact, that ANY anchor is available before the target element, makes the target navigable. If you don't use an anchor before the target element, the element is not navigable anymore (try this by selecting "skip to another content". It already starts at the top of the page, when navigating via TAB.

In Firefox this solution also works, the only thing that is not really beautiful is, that the "hidden" anchor element will also be highlighted during navigation with TAB. That results in an extra but unnecessary step during TAB navigation. This is only because of the HREF in the anchor (that is necessary for IE 5.5., 6, 7.

This solution is no effect in Chrome. It still not works there.
